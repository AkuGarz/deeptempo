# Contributing to DeepTempo

Thanks for considering contributing. This is a side project maintained by one person, so patience is appreciated.

## Ground rules

- **Keep PRs small and focused.** A PR that changes 800 lines across 14 files is hard to review. Break it up.
- **Include tests.** If you add a feature, add a test. If you fix a bug, add a regression test.
- **Don't add dependencies lightly.** Every new dependency is maintenance burden. Convince me it's worth it.
- **Respect the roadmap.** Check `docs/roadmap.md` to see what's planned. If you want to work on something that's not there, open an issue first.
- **This is Apache 2.0.** Your contributions will be under the same license.
